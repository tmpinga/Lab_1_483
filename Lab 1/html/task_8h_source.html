1161 /*
1162  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS ONLY
1163  * INTENDED FOR USE WHEN IMPLEMENTING A PORT OF THE SCHEDULER AND IS
1164  * AN INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
1165  *
1166  * Called from the real time kernel tick (either preemptive or cooperative),

1168  * for a finite period required removing from a blocked list and placing on
1169  * a ready list.

1173 /*
1174  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
1175  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
1176  *
1177  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
1178  *
1179  * Removes the calling task from the ready list and places it both
1180  * on the list of tasks waiting for a particular event, and the
1181  * list of delayed tasks. The task will be removed from both lists
1182  * and replaced on the ready list should either the event occur (and
1183  * there be no higher priority tasks waiting on the same event) or
1184  * the delay period expires.
1185  *
1186  * @param pxEventList The list containing tasks that are blocked waiting
1187  * for the event to occur.
1188  *
1189  * @param xTicksToWait The maximum amount of time that the task should wait
1190  * for the event to occur. This is specified in kernel ticks,the constant
1191  * portTICK_RATE_MS can be used to convert kernel ticks into a real time
1192  * period.
1193  */
1194 void vTaskPlaceOnEventList( const xList * const pxEventList, portTickType xTicksToWait ) PRIVILEGED_FUNCTION;
